# 产品需求文档（PRD）：AI故事图文创作助手

---

## 1. 产品概述

**产品名称**：StoryCraft AI（故事工坊）  
**产品定位**：面向自媒体创作者的AI故事结构化创作与多平台发布工具  
**目标用户**：公众号运营者、小红书博主、短篇故事创作者  
**核心价值**：将"热点洞察→故事创作→结构化分镜脚本（基因）→Agent适配出图→多平台发布"全流程压缩至分钟级。**用户掌握"基因"（结构化提示词），自由选择"成长环境"（生图工具）。**

**核心理念**：
> 文本是一种序列的约定——StoryCraft 负责产出高质量的结构化"基因"（故事 + YAML/JSON 分镜脚本 + 生图提示词），生图环节做成可插拔的 Agent 适配层，用户选择自己顺手的工具（Gemini、豆包、Qwen、Nano Banana、即梦等）去出图。

---

## 2. 用户画像细分

| 用户类型 | 特征 | 核心痛点 | 使用场景 |
|---------|------|---------|---------|
| **效率型博主** | 日更/高频更新，追求产量 | 灵感枯竭、制作耗时 | 每日批量生成3-5条短内容 |
| **质量型创作者** | 周更，追求精品爆款 | 热点把握不准、图文质量参差 | 基于热点深度创作1条爆款 |
| **新手入门者** | 刚起步，无设计基础 | 不会写故事、不会写生图提示词 | 模板化快速上手，降低门槛 |
| **矩阵运营者** | 管理多个账号 | 多平台适配繁琐、重复劳动 | 一次创作，多平台自动适配发布 |

## 3. 功能需求

### 3.1 核心功能模块

#### 模块A：热点雷达（故事灵感引擎）

| 功能点 | 需求描述 | 优先级 | 备注 |
|-------|---------|--------|------|
| A1 全网热点聚合 | 抓取微博热搜、抖音热榜、知乎热问、小红书趋势、公众号爆文 | P0 | 支持按领域筛选（情感/职场/育儿/宠物等） |
| A2 热点-故事关联推荐 | AI分析热点事件，推荐3-5个故事切入角度 | P0 | 例如：热点"职场35岁危机"→推荐"逆袭故事/温情故事/讽刺故事" |
| A3 热点时效预警 | 标记热点生命周期（萌芽期/爆发期/衰退期），建议最佳发布窗口 | P1 | 避免追"冷热点" |
| A4 历史爆款库 | 收录各平台高互动故事图文，支持按标签检索 | P1 | 用于学习和模仿 |
| A5 自定义监控 | 用户可设置关键词，自动推送相关热点 | P2 | 如监控"猫咪"、"副业"等 |
| A6 平台算法洞察 | 追踪各平台推荐算法偏好变化（公众号推荐流/小红书CES评分等），给出内容优化建议 | P2 | 如标题关键词、互动引导策略、最佳发布时段 |

#### 模块B：AI故事工坊

| 功能点 | 需求描述 | 优先级 | 备注 |
|-------|---------|--------|------|
| B1 故事生成 | 基于热点或自定义主题，生成完整短篇故事（300-2000字可调） | P0 | 支持风格选择：温情/悬疑/搞笑/治愈/励志 |
| B2 故事扩写 | 输入一句话梗概，AI扩展为完整故事 | P0 | 支持调整篇幅和细节密度 |
| B3 分镜脚本生成（核心） | 将故事拆解为3-8个图文分镜，每镜输出结构化描述（场景/动作/表情/细节/文案），支持YAML/JSON格式 | P0 | **核心差异化功能**。输出格式参考行业最佳实践（如公众号四格漫画的YAML分镜结构） |
| B4 多风格切换 | 同一故事可快速切换：口语化/文学化/网感化/治愈系 | P1 | 适配不同平台调性 |
| B5 标题优化 | 生成5个备选标题，含平台适配建议（小红书偏emoji/悬念，公众号偏深度），并标注推荐算法友好度 | P1 | A/B测试数据反馈优化 |
| B6 故事续写/改写 | 对生成内容局部修改、续写或改写结局 | P1 | 人机协作精修 |
| B7 角色一致性管理 | 定义角色档案（名称/外貌/性格），自动注入所有分镜脚本，确保跨图人物连贯 | P0 | 生图提示词中角色描述的一致性保障 |

#### 模块C：提示词工坊 + Agent适配层

| 功能点 | 需求描述 | 优先级 | 备注 |
|-------|---------|--------|------|
| C1 结构化生图提示词生成 | 基于分镜脚本，自动生成结构化生图提示词（包含角色外貌、场景、动作、表情、风格、色调、画幅比例等字段） | P0 | 默认输出YAML/JSON格式，支持自定义字段 |
| C2 Agent适配导出 | 一键将提示词转译为不同平台的输入格式——Gemini Imagen prompt、DALL-E 3 prompt、豆包/即梦/Nano Banana的文字指令、ComfyUI workflow等 | P0 | 内置适配模板，用户可自定义新增 |
| C3 风格预设管理 | 提供20+视觉风格预设（水彩/扁平插画/3D/国潮/复古/像素/手绘简笔画等），自动注入提示词的style/color_scheme字段 | P1 | 用户可收藏、自定义风格模板 |
| C4 内置API生图（可选） | 对已有官方API的模型（DALL-E 3、Gemini Imagen、Qwen等），支持配置API Key后直接在平台内生图 | P1 | 与C2手动导出互为补充，不是必选路径 |
| C5 智能排版 | 自动将图片和文案组合为平台适配的版式 | P1 | 小红书3:4/4:3，公众号长图/条漫 |
| C6 图文混排编辑器 | 可视化编辑器，拖拽调整图文位置、字体、配色 | P1 | 类Canva的简易操作 |
| C7 品牌水印 | 自动添加账号Logo/二维码/品牌色 | P2 | 防盗图+引流 |
| C8 批量提示词生成 | 一次生成整套故事的6-9组分镜提示词 | P1 | 效率优化 |

#### 模块D：发布中心

| 功能点 | 需求描述 | 优先级 | 备注 |
|-------|---------|--------|------|
| D1 多平台适配 | 自动将内容转译为各平台格式 | P0 | 小红书：标题+9图+话题标签；公众号：长图文+封面+摘要 |
| D2 一键发布 | 授权后自动发布到公众号（官方API）、小红书（RPA/官方API待定） | P1 | 公众号优先，小红书视API获取情况 |
| D3 定时发布 | 设置未来时间点自动发布 | P1 | 配合热点最佳窗口 |
| D4 发布预览 | 模拟各平台前端展示效果 | P1 | 避免格式错乱 |
| D5 草稿箱管理 | 保存未发布内容，支持标签分类 | P1 | 内容资产管理 |
| D6 发布状态追踪 | 记录已发布内容的状态（成功/失败/审核中） | P2 | 失败时重试/告警 |

### 3.2 扩展功能（差异化竞争力）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **提示词市场** | 用户可分享/交易优质的结构化提示词模板和风格预设 | P2 |

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 故事生成<10秒；分镜脚本+提示词生成<15秒；整套内容（故事→分镜→提示词）生成<1分钟 |
| **稳定性** | 核心功能可用性99.5%；生成失败自动重试并告警 |
| **安全性** | 用户数据隔离；内容审核过滤敏感信息（生成后+发布前双重审核）；符合各平台发布规范 |
| **扩展性** | 提示词模板和Agent适配器可热插拔；新平台接入周期<2周 |
| **成本** | 提供按量计费/订阅制/免费额度三档，AI文本生成成本可控（仅调用LLM，不含生图API成本） |
| **降级方案** | AI服务不可用时：排队重试 → 异步通知 → 模型自动切换备用方案；用户可导出提示词后手动去生图工具完成 |

## 5. 用户旅程（核心流程）

```
[发现热点] → [选择角度] → [生成故事] → [分镜拆解→结构化脚本] → [提示词生成] → [Agent适配导出/内置生图]
       ↑                                                                              ↓
       |                                                          [手动粘贴到豆包/Gemini等出图]
       |                                                                              ↓
       |                                                              [回传图片→排版设计]
       |                                                                              ↓
[数据回流优化] ←────────────────────────────────────────── [平台适配] ← [预览确认] ← [发布]
```

**典型场景示例**：
1. 用户早上看到"年轻人整顿职场"热搜
2. 系统推荐3个角度，用户选"搞笑整顿"
3. AI 生成800字故事 + 自动拆解为6格分镜脚本（YAML格式，含角色描述+每格场景/动作/表情/文案）
4. 用户选"手绘简笔画"风格，系统自动将风格参数注入提示词
5. 一键导出为 Nano Banana / Gemini Imagen / 豆包 可用的提示词文本
6. 用户在生图工具中出图后，回传图片到 StoryCraft 排版
7. 系统自动转译为小红书格式和公众号格式
8. 定时下午6点自动发布到公众号（小红书手动/RPA发布）

## 6. 竞品分析

| 竞品 | 优势 | 劣势 | 我们的机会 |
|------|------|------|-----------|
| **Coze + Nano Banana 工作流** | 低代码搭建，门槛极低，已有大量教程；直发公众号草稿箱 | 提示词结构深度不足，角色一致性控制弱；平台锁定（依赖Coze生态） | 更专业的分镜脚本结构化能力；多Agent适配，不锁定单一工具 |
| **Claude Code Skill（漫画工厂）** | 开发者友好，灵活度高，可深度定制 | 需要技术背景，无GUI，无多平台发布能力 | 面向非技术用户的图形化界面 + 一键多平台适配 |
| **n8n 自动化工作流** | 开源，可自建，灵活编排 | 搭建成本高，无内置故事创作AI，无多平台适配 | 内置故事AI + 分镜引擎 + 发布中心，开箱即用 |
| **ChatGPT + 手动生图工具** | 自由组合，质量上限高 | 流程割裂，提示词靠人工编写，无结构化分镜管理 | 结构化分镜脚本自动生成 + 角色一致性管理 |
| **Canva / 稿定设计** | 设计能力强，模板丰富 | 无AI故事生成能力，无分镜脚本管理 | 内容+分镜+设计一体化 |
| **各类AI写作工具** | 文本生成强 | 无结构化分镜输出，无配图提示词，无发布 | 全链路覆盖：故事→分镜脚本→Agent适配→发布 |

## 7. 技术架构建议

```
前端：React / Vue3（Web编辑器）+ 小程序 / H5（移动端）
后端：Node.js / Python

AI层：
  - 故事生成 + 分镜拆解：GPT-4 / Claude / Qwen / 自研微调模型
  - 结构化提示词引擎：自研模板引擎，输出YAML/JSON格式

生图适配层（可插拔Agent架构）：
  ┌─────────────────────────────────────────────┐
  │          Agent Adapter Manager               │
  ├──────┬──────┬──────┬──────┬──────┬──────────┤
  │Gemini│DALL·E│ Qwen │ 豆包  │ Nano │ 自定义   │
  │Imagen│  3   │Imagen│提示词 │Banana│ Agent    │
  │ API  │ API  │ API  │ 导出  │ 导出 │ 配置     │
  └──────┴──────┴──────┴──────┴──────┴──────────┘
  内置API调用（可选）│── 手动粘贴导出 ──│
                    排版回传（手动上传出图结果）

发布层：
  - 公众号：微信官方API（内容发布）
  - 小红书：优先申请官方API，备选RPA方案（模拟操作+内容发布）
  - 抖音图文 / 微博：后续扩展

数据层：
  - 热点爬虫（微博/抖音/知乎/小红书/公众号）
  - 用户内容库 + 提示词模板库
  - 公众号后台数据（官方API可获取阅读/分享/互动数据）
```

## 9. 成功指标（KPI）

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单内容生产时间（故事→分镜脚本→提示词导出） | <5分钟 | 不含手动出图时间 |

| 提示词导出后出图可用率 | >75% | 导出的提示词在主流生图工具中一次出图可用的比例 |

## 10. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 小红书/抖音无官方发布API | 自动化发布受阻 | 公众号优先（有官方API）；小红书备选RPA方案 + 手动发布引导；持续关注官方API开放动态 |
| 生图工具迭代快，API/能力频繁变化 | Agent适配器需持续维护 | 适配层抽象为插件架构，社区贡献+官方维护热门适配器 |
| AI生成内容同质化 | 用户内容雷同，平台降权 | 强化个性化prompt注入（角色库、风格偏好）；热点时效性确保内容差异化 |
| 公众号推荐算法变化 | 流量依赖推荐，策略失效 | 多平台分发降低单平台依赖；A6平台算法洞察持续追踪 |
| 版权争议 | 法律风险 | 内置版权检测 + 用户协议明确权责 + 提示词市场引入原创认证 |
| AI服务不可用 | 核心功能中断 | 多模型备份自动切换 + 降级方案（导出提示词后用户可离线使用） |
| 竞品（Coze/n8n工作流）快速迭代 | 用户被低代码方案分流 | 聚焦分镜脚本结构化深度和角色一致性——这是低代码工作流做不到的 |
